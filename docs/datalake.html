datalake – Manage CDP Datalakes¶

  • Synopsis

  • Requirements

  • Parameters

  • Examples

  • Return Values

  • Status

Synopsis¶

  • Create and delete CDP Datalakes.

  • To start and stop a datalake, use the env module to change the associated CDP Environment’s state.

Requirements¶

The below requirements are needed on the host that executes this module.

  • cdpy

Parameters¶

Parameter

Choices/Defaults

Comments

name


str


Required

The name of the datalake. This name must be unique, must have between 5 and 100 characters, and must contain only lowercase letters, numbers, and hyphens. Names are case-sensitive.


Aliases: datalake

state
Choices:

  • present ↞

  • absent

The declarative state of the datalake. If creating a datalake, the associate environment must be started as well.

environment
Choices:

  • env

The CDP environment name or CRN to which the datalake will be attached. If the environment is AWS-based, instance_profile and storage must be present.

instance_profile


str

(AWS) The IAM instance profile of the ID Broker role, which can assume the Datalake Admin S3 role. (Azure) The URI of the Identity of the ID Broker Role, which can assume the Datalake Admin ADLS role. (GCP) The Service Account email of the ID Broker Role, which can assume the Datalake Admin GCS role.

storage


str

(AWS) The S3 bucket (and optional path) for the Storage Location Base for the datalake, starting with s3a:// (Azure) The ADLS bucket URI (and optional path) for the Datalake storage (GCP) The bucket name and optional path for the GCS Storage Location Base for the Datalake, starting with gs://


Aliases: storage_location, storage_location_base

runtime


str

The Cloudera Runtime version for the datalake, when supported

scale
Choices:

  • LIGHT_DUTY ↞

  • MEDIUM_DUTY_HA

The scale of the datalake.

tags


dict

Tags associated with the datalake and its resources.


Aliases: datalake_tags

force


bool

Flag indicating if the datalake should be force deleted. This option can be used when cluster deletion fails. This removes the entry from Cloudera Datalake service. Any lingering resources have to be deleted from the cloud provider manually.

wait


bool

Flag to enable internal polling to wait for the datalake to achieve the declared state. If set to FALSE, the module will return immediately.

delay


int

The internal polling interval (in seconds) while the module waits for the datalake to reach the declared state.


Aliases: polling_delay

timeout


int

The internal polling timeout (in seconds) while the module waits for the datalake to achieve the declared state.


Aliases: polling_timeout

verify_tls


bool

Verify the TLS certificates for the CDP endpoint.


Aliases: tls

debug


bool

Capture the CDP SDK debug log.


Aliases: debug_endpoints

profile


str

If provided, the CDP SDK will use this value as its profile.

Examples¶

# Note: These examples do not set authentication details.

# Create a datalake in AWS
- cloudera.cloud.datalake:
    name: example-datalake
    state: present
    environment: an-aws-environment-name-or-crn
    instance_profile: arn:aws:iam::1111104421142:instance-profile/example-role
    storage: s3a://example-bucket/datalake/data
    tags:
      project: Arbitrary content

# Create a datalake in AWS, but don't wait for completion (see datalake_info for datalake status)
- cloudera.cloud.datalake:
    name: example-datalake
    state: present
    wait: no
    environment: an-aws-environment-name-or-crn
    instance_profile: arn:aws:iam::1111104421142:instance-profile/example-role
    storage: s3a://example-bucket/datalake/data
    tags:
      project: Arbitrary content

# Delete the datalake (and wait for status change)
  cloudera.cloud.datalake:
    name: example-datalake
    state: absent
