env – Manage CDP Environments¶

  • Synopsis

  • Requirements

  • Parameters

  • Examples

  • Return Values

  • Status

Synopsis¶

  • Create, update, and delete CDP Environments

  • Note that changing states, in particular, creating a new environment, can take several minutes.

Requirements¶

The below requirements are needed on the host that executes this module.

  • cdpy

Parameters¶

Parameter

Choices/Defaults

Comments

name


str


Required

The name of the target environment. Names must begin with a lowercase alphanumeric, contain only lowercase alphanumerics and hyphens, and be between 5 to 28 characters in length.


Aliases: environment

state


str
Choices:

  • present ↞

  • started

  • stopped

  • absent

The declarative state of the environment If state=present, one of cloud or credential must be present.

cloud


str
Choices:

  • aws

  • azure

  • gcp

The cloud provider or platform for the environment. Requires region, credential, log_location, and log_identity. If cloud=aws, one of public_key or public_key_id must be present. If cloud=aws, one of network_cidr or vpc_id must be present. If cloud=aws, one of network_cidr or vpc_id must be present. If cloud=aws, one of inbound_cidr or default_sg and knox_sg must be present.

region


str

The cloud platform specified region

credential


str

The CDP credential associated with the environment

project


str

Name of Project when deploying environment on GCP

inbound_cidr


str

CIDR range which is allowed for inbound traffic. Either IPv4 or IPv6 is allowed. Mutually exclusive with default_sg and knox_sg.


Aliases: security_cidr

default_sg


str

Security group where all other hosts are placed. Mutually exclusive with inbound_cidr.


Aliases: default, default_security_group

knox_sg


str

Security group where Knox-enabled hosts are placed. Mutually exclusive with inbound_cidr.


Aliases: knox, knox_security_group

public_key_text


str

The content of a public SSH key. Mutually exclusive with public_key_id.


Aliases: ssh_key_text

public_key_id


str

The public SSH key ID already registered in the cloud provider. Mutually exclusive with public_key_text.


Aliases: public_key, ssh_key, ssh_key_id

log_location


str

(AWS) The base location to store logs in S3. This should be an s3a:// url.


Aliases: storage_location_base

log_identity


str

(AWS) The instance profile ARN assigned the necessary permissions to access the S3 storage location, i.e. log_location.


Aliases: instance_profile

network_cidr


str

(AWS) The network CIDR. This will create a VPC along with subnets in multiple Availability Zones. Mutually exclusive with vpc_id and subnet_ids.

vpc_id


str

(AWS) The VPC ID. Mutually exclusive with network_cidr and requires subnet_ids.


Aliases: vpc

subnet_ids


list

(AWS) One or more subnet identifiers within the VPC. Mutually exclusive with network_cidr and requires vpc_id.


Aliases: subnets

tags


dict

Tags associated with the environment and its resources.


Aliases: environment_tags

workload_analytics


bool

Flag to enable diagnostic information about job and query execution to be sent to Workload Manager for Data Hub clusters created within the environment.

description


str

A description for the environment.


Aliases: desc

tunnel


bool

Flag to enable SSH tunnelling for the environment.


Aliases: enable_tunnel, ssh_tunnel

freeipa


dict

The FreeIPA service for the environment.

proxy


str

The name of the proxy config to use for the environment.


Aliases: proxy_config, proxy_config_name

wait


bool

Flag to enable internal polling to wait for the environment to achieve the declared state. If set to FALSE, the module will return immediately.

delay


int

The internal polling interval (in seconds) while the module waits for the environment to achieve the declared state.


Aliases: polling_delay

timeout


int

The internal polling timeout (in seconds) while the module waits for the environment to achieve the declared state.


Aliases: polling_timeout

s3_guard_name


str

(AWS) AWS Dynamo table name for S3 Guard.


Aliases: s3_guard, s3_guard_table_name

verify_tls


bool

Verify the TLS certificates for the CDP endpoint.


Aliases: tls

debug


bool

Capture the CDP SDK debug log.


Aliases: debug_endpoints

profile


str

If provided, the CDP SDK will use this value as its profile.

Examples¶

# Note: These examples do not set authentication details.

# Create an environment
- cloudera.cloud.env:
    name: example-environment
    state: present
    credential: example-credential
    cloud: aws
    region: us-east-1
    log_location: s3a://example-bucket/datalake/logs
    log_identity: arn:aws:iam::981304421142:instance-profile/example-log-role
    public_key_id: example-sshkey
    network_cidr: 10.10.0.0/16
    inbound_cidr: 0.0.0.0/0
    tags:
      project: Arbitrary content

# Create an environment, but don't wait for completion (see env_info)
- cloudera.cloud.env:
    name: example-environment
    state: present
    wait: no
    credential: example-credential
    cloud: aws
    region: us-east-1
    log_location: s3a://example-bucket/datalake/logs
    log_identity: arn:aws:iam::981304421142:instance-profile/example-log-role
    public_key_id: example-sshkey
    network_cidr: 10.10.0.0/16
    inbound_cidr: 0.0.0.0/0
    tags:
      project: Arbitrary content

# Update the environment's CDP credential
- cloudera.cloud.env:
    name: example-module
    credential: another-credential

# Stop the environment (and wait for status change)
- cloudera.cloud.env:
    name: example-module
    state: stopped

# Start the environment (and wait for status change)
- cloudera.cloud.env:
    name: example-module
    state: started

# Delete the environment (and wait for status change)
  cloudera.cloud.env:
    name: example-module
    state: absent
