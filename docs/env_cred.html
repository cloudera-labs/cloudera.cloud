env_cred – Create, update, and destroy CDP credentials¶

  • Synopsis

  • Requirements

  • Parameters

  • Examples

  • Return Values

  • Status

Synopsis¶

  • Create, update, and destroy CDP credentials.

  • The module support check_mode.

Requirements¶

The below requirements are needed on the host that executes this module.

  • cdpy

Parameters¶

Parameter

Choices/Defaults

Comments

name


str


Required

The name of the Credential. The name must conform to the CDP Credential format, which is lowercase letters, numbers, and hyphens only.


Aliases: credential

state
Choices:

  • present ↞

  • absent

Establish the state of the Credential in CDP.

cloud
Choices:

  • aws

  • azure

  • gcp

The target cloud provider for the Credential. Required if state=present.

role


str

The CDP cross-account role for AWS For cloud=aws, this is the Role ARN for the cross-account role.


Aliases: arn, role_arn

subscription


str

The Subscription ID or URI of the Azure Subscription being used

tenant


str

The URI of the Azure Tenant

application


str

The ApplicationId of the Azure Application used for access

secret


str

The Secret for the Application access on Azure The path to the Key File for the Service Account being used on Google

description


str

Descriptive text for the Credential.


Aliases: desc

retries


int

Number of times to retry the create operation if a possible eventual consistency error is returned Set to 0 to fail immediately on such errors

delay


int

Delay period in seconds between retries

verify_tls


bool

Verify the TLS certificates for the CDP endpoint.


Aliases: tls

debug


bool

Capture the CDP SDK debug log.


Aliases: debug_endpoints

profile


str

If provided, the CDP SDK will use this value as its profile.

Examples¶

# Note: These examples do not set authentication details.

# Create a CDP Credential for AWS
- cloudera.cloud.env_cred:
    state: present
    cloud: aws
    name: example-credential
    description: This is an example Credential
    role: arn:aws:iam::981304421142:role/some-cross-account-role

# Delete a CDP Credential
- cloudera.cloud.env_cred:
    state: absent
    name: example-credential

# Create a CDP Credential for AWS and log the output of the CDP SDK in the return values
- cloudera.cloud.env_cred:
    name: example-credential
    debug: yes
