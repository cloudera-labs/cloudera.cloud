dw_cluster – Create or Delete CDP Data Warehouse Clusters¶

  • Synopsis

  • Requirements

  • Parameters

  • Examples

  • Return Values

  • Status

Synopsis¶

  • Create or Delete CDP Data Warehouse Clusters

Requirements¶

The below requirements are needed on the host that executes this module.

  • cdpy

Parameters¶

Parameter

Choices/Defaults

Comments

id


str


Required

If an ID is provided, that Data Warehouse Cluster will be deleted if state=absent


Aliases: name

env


str


Required

The name of the target environment


Aliases: environment, env_crn

aws_public_subnets


list


Required

List of zero or more Public AWS Subnet IDs to deploy to

aws_private_subnets


list


Required

List of zero or more Private AWS Subnet IDs to deploy to

az_subnet


str


Required

Azure Subnet Name, not URI

az_enable_az


bool


Required

Whether to enable AZ mode or not

state
Choices:

  • present ↞

  • absent

The declarative state of the Data Warehouse Cluster

wait


bool

Flag to enable internal polling to wait for the Data Warehouse Cluster to achieve the declared state. If set to FALSE, the module will return immediately.

delay


int

The internal polling interval (in seconds) while the module waits for the Data Warehouse Cluster to achieve the declared state.


Aliases: polling_delay

timeout


int

The internal polling timeout (in seconds) while the module waits for the Data Warehouse Cluster to achieve the declared state.


Aliases: polling_timeout

verify_tls


bool

Verify the TLS certificates for the CDP endpoint.


Aliases: tls

debug


bool

Capture the CDP SDK debug log.


Aliases: debug_endpoints

profile


str

If provided, the CDP SDK will use this value as its profile.

Examples¶

# Note: These examples do not set authentication details.

# Delete a Data Warehouse Cluster
- cloudera.cloud.dw_cluster:
    state: absent
    id: my-id

# Request Azure Cluster creation
- cloudera.cloud.dw_cluster:
    env_crn: crn:cdp:environments...
    az_subnet: my-subnet-name
    az_enable_az: yes

# Request AWS Cluster Creation
- cloudera.cloud.dw_cluster:
    env_crn: crn:cdp:environments...
    aws_public_subnets: [subnet-id-1, subnet-id-2]
    aws_private_subnets: [subnet-id-3, subnet-id-4]
